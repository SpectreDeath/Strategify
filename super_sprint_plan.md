# Strategify Super Sprint Plan: The "Cognitive Core" Update

This document provides a highly detailed execution plan designed for a coding sub-agent. The primary objective of this sprint is to integrate previously established components—specifically, the Large Language Model (LLM) decision engines, the Prolog-based Theory of Mind (ToM) epistemology, and the Clojure-based timeline branching algorithms—into a unified, hyper-intelligent agent architecture.

By following this sprint, the agent will evolve the project from simple Nash equilibrium loops into a deeply cognitive simulation where agents form beliefs, evaluate counterfactual realities, and execute grand strategies.

---

## Phase 15: LLM Orchestration & Symbolic Grounding
**Goal:** We need the LLM to make decisions not just based on raw numbers, but based on the rich, logical context provided by our Prolog and Clojure bridges.

### Task 1: Epistemic Prompt Injection
- **Target File:** `strategify/reasoning/llm.py`
- **Action:** Modify the `_build_prompt` function to inject *Theory of Mind* data. 
- **Details:** Before querying the LLM, call the `StrategicBridge` (from `strategify.logic.bridge`) for the acting agent. Retrieve lists of what the agent `knows()` and what the agent `believes()`. Add a new section to the prompt: `"Epistemology (Your Beliefs & Knowledge)"`. This grounds the LLM in the agent's subjective reality, not the simulation's objective reality.

### Task 2: Clojure Counterfactual Forecaster
- **Target File:** `strategify/reasoning/llm.py`
- **Action:** Inject future state predictions.
- **Details:** Import the `ClojureBridge`. For the current `state`, use `branch_timelines()` to generate the immediate next states for available moves (e.g., `["Escalate", "Deescalate"]`). Append these abbreviated predicted JSON states to the LLM prompt under `"Counterfactual Futures"`. The LLM should now decide its action based on Lisp-simulated futures.

### Task 3: The `CognitiveActorAgent`
- **Target File:** `strategify/agents/state_actor.py` (or create `cognitive_actor.py`)
- **Action:** Create a subclass of `StateActorAgent` named `CognitiveActorAgent`.
- **Details:** Override the `decide()` method. Instead of using Python matrix addition, this agent purely delegates to `LLMDecisionEngine.query()`, passing the rich state packet generated by Tasks 1 and 2.

## Phase 16: Prolog Deep Epistemology & Deception
**Goal:** Expand `traits.pl` and use standard Prolog unification to model disinformation, espionage, and nested beliefs (A believes B knows X).

### Task 1: Disinformation Mechanics
- **Target File:** `strategify/logic/traits.pl`
- **Action:** Add rules for deception.
- **Details:** Implement a rule `believe(Agent, Fact) :- is_propaganda(Fact), personality(Agent, gullible).` and `know(Agent, Fact) :- verify(Fact), personality(Agent, analyst).` 

### Task 2: Propagating Facts from Geography to Logic
- **Target File:** `strategify/sim/model.py`
- **Action:** Connect the Python spatial grid to the Prolog Knowledge Base.
- **Details:** Create a global `StrategicBridge` in `GeopolModel`. Every tick in `step()`, translate major world events (e.g., "Alpha invaded Bravo") into Prolog facts (`assertz(event(invade, alpha, bravo))`). 

### Task 3: Information Warfare Action
- **Target File:** `strategify/game_theory/crisis_games.py`
- **Action:** Add a `disinformation_game`.
- **Details:** Actions: `["SpreadFakeNews", "Censor"]`. If an agent picks "SpreadFakeNews", assert a false fact into the Prolog engine. Agents with high "Economic Bias" or specific personalities will adopt the false belief, skewing their next LLM prompt.

## Phase 17: Clojure Monte Carlo Tree Search (MCTS)
**Goal:** Turn our Clojure-based timeline execution into an asynchronous simulation engine capable of predicting hundreds of steps ahead to cull poor strategies.

### Task 1: Expand the Clojure State Map
- **Target File:** `strategify-clj/src/strategify/core.clj` (Note: This is the external Clojure service directory).
- **Action:** Extend the Lisp state map.
- **Details:** The current model just tracks `:hawk`/`:dove` payloads. Add `:economic-reserve`, `:stability`, and `:un-resolutions` to the core game loop in Lisp. 

### Task 2: Fast Path Combat Resolution
- **Target File:** `strategify/sim/conflict.py`
- **Action:** Offload combat math to Clojure.
- **Details:** When `_execute_combat()` triggers between two massive armies, instead of evaluating iterative damage in Python, package the `strength`, `readiness`, and `terrain` into a JSON payload. Send it via `ClojureBridge` to a new `(resolve-combat state)` function to instantly return the probabilistic outcome of a 1-month campaign, significantly optimizing the kinetic phase.

## Phase 18: The Grand Strategy Tournament
**Goal:** Run a headless tournament to evaluate the supremacy of these cognitive architectures.

### Task 1: Tournament Runner
- **Target File:** `scripts/tournament_runner.py` (New File)
- **Action:** Create a script to instantiate `GeopolModel` repeatedly.
- **Details:** Run 50 simulations using `StateActorAgent` (Nash default). Run 50 using `CognitiveActorAgent` (LLM + Prolog + Lisp). Use `DataCollector` to track global average `stability` and `global_tension`.
- **Expected Outcome:** Prove that the LLM/Prolog logic agents avoid catastrophic nuclear/kinetic escalations significantly better than the raw game-theoretic agents by utilizing the Theory of Mind and timeline prediction engines.
